隐私政策

最后更新:2026 年 5 月

1. 我们收集什么

当你使用 Strava 账号登录后,本服务会从 Strava API 读取并存储以下数据:

  • 个人资料:姓名、Strava 用户 ID、头像、性别、体重(如果你在 Strava 中设置过)、邮箱。
  • 骑行活动数据:日期、距离、时长、爬升、配速、平均/最大功率、心率、踏频。
  • 每次骑行的逐秒数据流:时间、距离、坐标、海拔、速度、心率、踏频、功率、温度。
  • 用于持续访问 Strava 的 OAuth refresh token 与 access token。

除了你在分析页面手动输入的年龄之外,本服务不收集任何其他个人信息。

2. 数据如何使用

  • 计算并展示你的功率画像、训练负荷、车手类型与比赛策略——仅供你本人查看。
  • 调用 Strava API 同步新的骑行数据。

我们不会:出售、分享或转让你的数据给任何第三方;用你的数据训练机器学习模型;在未经你同意的情况下公开展示你的活动。

3. 数据存储

所有数据存放在我们部署在 Fly.io 服务器上的 SQLite 数据库与文件存储中。该服务器位于香港(hkg)区域,数据不会同步到其他区域。

4. 数据删除

你可以随时通过下列任一方式删除你的全部数据:

  • 登录后进入 账号设置 页面,点击"删除我的账号"。本服务会立即:
    1. 调用 Strava 的 /oauth/deauthorize 撤销本应用的访问权限。
    2. 从服务器删除你的所有骑行数据、数据流文件、用户资料、登录会话。
  • 在 Strava 应用列表中(strava.com/settings/apps)取消授权。Strava 会通过 webhook 通知本服务,我们会在 24 小时内自动删除你的数据。

5. Cookie

本服务仅使用一个会话 cookie(coach_sid),用于记住你的登录状态。该 cookie 仅在 HTTPS 下传输、对 JavaScript 不可见(HttpOnly)、有效期 30 天。不使用任何第三方 / 分析 / 广告 cookie。

6. 第三方

本服务通过 Strava API v3 获取你的活动数据,你在登录时所授予的权限决定了我们能读取的内容。请同时阅读 Strava 隐私政策。

7. 联系

关于本政策的疑问可以通过 GitHub Issues 联系。
